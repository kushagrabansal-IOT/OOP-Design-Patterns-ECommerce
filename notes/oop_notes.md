# Design Patterns Reference

## Creational Patterns
Singleton: One instance. Use for DB conn, config, logger.
Factory: Create objects without specifying class. Use when type varies at runtime.
Builder: Step-by-step complex object. Use when many optional parameters.

## Structural Patterns
Decorator: Add behavior without modifying class. Use for middleware.
Adapter: Wrap incompatible interfaces.
Proxy: Control access to object.

## Behavioral Patterns
Strategy: Swap algorithms. Use for payment, sorting, discount.
Observer: Event subscription. Use for notifications, event bus.
Command: Encapsulate request. Use for undo/redo.
